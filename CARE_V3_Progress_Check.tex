\begin{frame}{The Challenge: Transitioning to ResNet-18}
    \textbf{Observed Limitations of Static Initialization:}
    \begin{itemize}
        \item Study-proven initializations (Kaiming/Xavier) only solve the problem at $T=0$.
        \item In deep SNNs, neurons that are active at $T=0$ can still "starve" and die during training due to weight decay or shifts in input distribution.
    \end{itemize}
    
    \vspace{0.3cm}
    \textbf{The Dynamic Starvation Problem:}
    \begin{enumerate}
        \item Standard deep models lose effective capacity as training progresses.
        \item Rare features (e.g., small objects in Cityscapes) are disproportionately affected by gradient starvation.
    \end{enumerate}
\end{frame}

\begin{frame}{CARE: Dynamic Capacity Insurance}
    \textbf{The Mechanical Breakthrough:}
    \begin{itemize}
        \item CARE is not just an initialization fix; it is a **continuous insurance policy** for representational capacity.
        \item By targeting the \textbf{BatchNorm Gamma}, CARE ensures that every filter in a ResNet block maintains a minimum "readiness" level throughout training.
    \end{itemize}
    
    \vspace{0.3cm}
    \textbf{Beyond Static Heuristics:}
    \begin{itemize}
        \item While standard homeostasis blindly forces activity, CARE’s **SNR-Gating** ensures that only neurons with coherent input are amplified.
        \item This prevents "Capacity Rot" where unused neurons are permanently zeroed out by supervised gradients.
    \end{itemize}
\end{frame}
